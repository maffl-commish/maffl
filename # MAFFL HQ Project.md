# MAFFL HQ Project

Build environment for MAFFL HQ — the web hub for the **Mid-Atlantic Fantasy Football League**
(MAFFL, founded 2002). This file is the canonical build spec: any Claude Code session should
read it first to orient.

> **Companion docs:** `CHANGE_INVENTORY.md` (every changeable datapoint, its authoritative
> source, mirrors, triggers, and the change-event calendar) and `AUDIT.md` (the 2026-06 data
> provenance audit). When this spec and the inventory ever disagree, the inventory is the more
> detailed source — update both.

---

## Current Direction (as of 2026-06)

The project is moving **from hand-maintained frozen HTML to a build pipeline.** Target: CSV
sources in `./data/` are the single source of truth; a build step (`build.py`) regenerates the
data embedded in each HTML page so the pages can never silently drift from the CSVs. Launch
target **July 1, 2026**; June is the build window, with no inbound data changes except the
commissioner's own.

This replaces the previous "every file hand-maintained, no build step" approach. The
self-contained-output rule (below) still holds — the build *inlines* data so the output files
remain dependency-free for Gmail distribution.

## Data Authority Rules

The governing principle: **one authoritative source per datapoint; everything else is a mirror
regenerated by the build.** When a source is itself wrong, fix the source, then rebuild — the
rule is *single source of truth*, not *the CSV is always right*.

- **Career totals** (Championships, Runner-Ups, Playoffs, Division Titles, W/L/T, Win %) →
  **recomputed from `cleaned_maffl_revised.csv`** year-by-year flags. The Owners Sheet is the
  *published mirror*, regenerated — not hand-typed. (Hand-typing is what produced the Tony Brooks
  division-title drift; recompute closes that class of error.)
- **Year-by-year flags & records** → `cleaned_maffl_revised.csv`.
- **Canonical team name per owner-season** → `MAFFL_Division_History_2005_2025.csv`.
- **Power ratings (OVR/Clutch/Grind/Heat) + scout notes** → **authored by the commissioner**,
  stored in the power-rankings source file (to create). Not derived; no recompute check.
- **Prize payouts** → **formula-derived** (pool = $100 × teams, 67/33 split, placement %s). The
  build can compute and validate them. Pay *status* is a manual input.
- **Credits** → see Credit Model below.

**Owner name normalization:** co-owner pairs are single units —
`Brian Murello / Ron Murello`, `Jon Murello / Rick Simmons`, `Marcus Ruby / Joe Ruby`. Never
split a pair's record. `Mike Murello` == `Michael Murello`. Normalization should live in **one
place in the build** (canonical = forward slash, spaces both sides — pending final confirmation;
six variant formats currently exist across files and must be standardized).

## Credit Model (Google Sheet RETIRED)

- The Google Sheet is **gone.** `Credit_Log.csv` (a transaction log in `./data/`) is the source
  of truth.
- **Balances are derived:** an owner's balance = sum of their Credit_Log rows. The balance column
  in the Owners Sheet is a **mirror**, regenerated — not hand-maintained.
- **Intake:** the Google Form (`https://forms.gle/57KKnEta1wfjzDNy9`) is a *request queue only*.
  Commissioner reviews each request; if approved, appends one row to `Credit_Log.csv`; rebuilds.
- The credit registry on credits.html is now source-backed (generated from the log) and verifiable.
- **One-time migration (June):** seed `Credit_Log.csv` from current known balances — decide
  opening-balance-per-owner vs. back-entered history at build time.

## Standing Exceptions (build validation must NOT flag these)

A validation pass should whitelist these confirmed-correct anomalies so it never re-raises
settled questions:

1. **2002 co-championship** — Mike Murello AND Josh Lavrinc. **25 championship flags across 24
   seasons is correct.** Any "one champion per year" assertion is wrong by design here.
2. **2002–2004 blank W/L/T** — founding era never tracked. Blank is correct.
3. **2007 blank draft prices** — permanent hard-drive loss. Blank is correct.
4. **"Michael Murello"** in the division file = Mike Murello (normalization handles it).
5. **Co-owner pairs are single units** — never split.
6. **Credit-packet "2026 League" column holds 2025 data** — known mislabel; resolve real 2026
   tier by 2026 Division letter (A–D = Upper, N/A = Lower), not that column.
7. **$2 Lower-Tier weekly-pool rounding** in 2025 prizes — disclosed in the CSV meta.

---

## Build Validation Gates (run before any publish)

- Championship flags total == 25 (2002 co-champ exception above).
- Every owner's recomputed W/L/T from year-by-year matches the published mirror.
- Win % == W/(W+L+T) to rounding.
- Every credit balance == sum of that owner's Credit_Log rows.
- Prize payouts sum to the computed pool.
- Draft pick count per team matches the era's roster size; no Kicker drafted 2025+.
- Every owner name in every file resolves to a canonical name.
- Power-ranking ranks are unique 1..N; every rated owner exists.

## Self-Contained Output Rule

Every distributed HTML file must be fully self-contained — all CSS, JS, and data inline. No CDN
calls. Only external dependency allowed: Google Fonts via `<link>`. The build pipeline *inlines*
generated data to preserve this for Gmail attachments. (iOS standalone PWA meta tags must appear
in every page's `<head>` — verified by `check-standalone.ps1`.)

## Change Events (see CHANGE_INVENTORY.md §5 for full detail)

1. **Season-End Push** — priority outputs first: **prizes + promotion/relegation** (the
   time-sensitive money-and-status results the league is waiting on), then credit awards, then
   stats / history / season count. Items can be done together; the only build-internal dependency
   is that stats *recompute from* history, so history rows must exist before stats regenerate.
2. **Power Rankings** — authored, later off-season, standalone.
3. **Division Alignment** — by August 1 (Upper-Tier).
4. **Draft Results** — bulk append after Labor Day.
5. **Weekly In-Season** — paste results → build → weekly + standalone.
6. **Ad-Hoc Correction** — fix the source file, never just the HTML; rebuild.
7. **Rule Change** — edit rules CSV; rebuild; announce.
8. **UI/Content** — pure code, no data.
